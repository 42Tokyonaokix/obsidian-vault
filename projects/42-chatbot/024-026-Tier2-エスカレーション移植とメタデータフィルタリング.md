## まとめ

feat/tier2-port-and-inquiry-form ブランチ上で進めた Tier 2 機能群の移植と拡張フェーズ。旧アーキテクチャ（ContextVar + HTMX/Jinja2）から新アーキテクチャ（SingleReact + RunnableConfig + React Widget）への移植が共通テーマ。

まず Tier 2 エスカレーション enrichment の移植（024）で、`feat/tier2-escalation-enrichment` ブランチの直接マージが16件のコンフリクトで断念となり、機能エッセンス（TAG_DEFINITIONS、business_tag 検出、手続き案内チャンク7件）を手動移植する方針を決定。mock-platform に問い合わせフォームページ（/inquiry）を新設し、Widget の全 bot メッセージに常時表示する問い合わせボタンからの遷移先とした。

続いて問い合わせフォームの高度化（025）で、カテゴリ選択ドロップダウン（8カテゴリ）と LLM 下書き自動生成機能を実装。`POST /api/v1/zendesk/draft` エンドポイントを新設し、Widget から thread_id 付きで遷移した際に会話ログからカテゴリ別の推奨項目+サマリを自動プリフィルする仕組みを構築した。

最後に RAG メタデータフィルタリング（026）を新アーキテクチャに移植。`single_react.py` の既存 config.metadata パイプラインを活用し、voltage_type/area の hard-exclude フィルタ（タグ付き不一致→除外、タグなし→通過）を `knowledge_search` ツールに適用。React Widget に FilterSelector コンポーネント（Shadow DOM 対応インラインスタイル）と Cookie 永続化フック（`useFilterCookie`）を追加した。Subagent-Driven Development で5タスクを実行し、各タスクに spec + quality の2段階レビューを実施。
