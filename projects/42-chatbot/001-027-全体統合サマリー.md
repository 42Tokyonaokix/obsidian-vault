## アーキテクチャ移行の経緯

本プロジェクトで最も重要な技術的判断は **旧アーキテクチャ → 新アーキテクチャへの移行** であった。

### 旧アーキテクチャ（`Naoki/cs_support` 系統）
- **状態管理**: Python ContextVar（暗黙的なスレッドローカル変数）
- **UI**: HTMX + Jinja2 サーバーサイドレンダリング
- **オーケストレーション**: MultiAgent（複数エージェント協調）

### 新アーキテクチャ（main 系統）
- **状態管理**: LangGraph RunnableConfig.metadata（明示的な設定伝播）
- **UI**: React Widget（Shadow DOM 対応、SSE ストリーミング）
- **オーケストレーション**: SingleReact（単一 ReAct エージェント）

### 移行で発生した問題

012（Tier2 enrichment）と 014（メタデータフィルタリング）は旧アーキテクチャベースで作成されたため、main への直接マージができなかった（16件のコンフリクト）。結果として 024 と 026 で手動移植が必要になり、同じ機能を2回実装する形となった。

この経験から、**新規機能は必ず新アーキテクチャ（SingleReact + RunnableConfig）ベースで開発する** という方針が確立された。
