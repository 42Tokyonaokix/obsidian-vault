## 概要

42-chatbot プロジェクトの現状・成果・今後のロードマップを整理した資料。明日のプレゼン用。

## 1. プロダクト概要

**42-chatbot** — エネルギー事業者向けエンタープライズ AI チャットボット

| 項目 | 内容 |
|------|------|
| 技術スタック | Python 3.12 / FastAPI / LangGraph / PostgreSQL + pgvector / React 19 |
| コア機能 | ReAct エージェント（7ツール）、プラガブル RAG 検索（4戦略）、SSE ストリーミング |
| マルチエージェント | Data Agent / Action Agent / Knowledge Agent の3エージェント構成 |
| セキュリティ | JWT RS256 認証、入出力ガードレール、CORS 制御 |
| インフラ | Docker Compose（ローカル）、Celery（文書取込パイプライン） |

## 3. Zendesk 問い合わせ分析結果

339件を5段階に分類し、ROI ベースで優先度を設定。

```
Tier 1 (44件/13%) — 制度説明      → ナレッジ追加で解決 ★最優先
Tier 2 (42件/12%) — 手続き系FAQ   → エスカレーション改善
Tier 3 (93件/27%) — 要機能開発    → A-1〜A-4 で対応
Tier 4 (72件/21%) — 個別対応必須  → 人間対応（スコープ外）
Tier 5 (88件/26%) — スコープ外    → 別システム/別チーム
```

### Tier 3 機能一覧（A-1〜A-4）

| ID | 機能 | Zendesk件数 | 優先度 |
|----|------|------------|--------|
| A-1 | 契約容量変更ナビゲーター | 34件 | HIGH |
| A-2 | 請求スケジュール・ステータス照会 | 20件 | HIGH |
| A-3 | 容量拠出負担金シミュレーション | 34件 | MEDIUM |
| A-4 | 解約・切替ウィザード | 33件 | MEDIUM |

## 5. 推奨ロードマップ

```
Week 1: Tier 1 ナレッジチャンク投入 + RAG フィルタリング
         → Zendesk 13% (44件) を即座にカバー
Week 2: Tier 2 エスカレーション改善 + A-2 請求スケジュール
         → さらに 12%+6% をカバー
Week 3: A-1 容量変更ナビゲーター
         → 最多問い合わせ (34件) に対応
Week 4: A-4 解約ウィザード + A-3 容量拠出シミュレーション
         → 残りの Tier 3 をカバー
```

**目標**: 4週間で Zendesk 問い合わせの約52%（176件/339件）を自動対応可能にする

## 6. 技術的な課題・注意点

1. **コードレビュー指摘事項**: APIキー露出、アップロードサイズ無制限、スレッド所有権チェック欠如（本番前に修正必須）
2. **cs_support → main 未移植**: エリアフィルタ、Action Agent、Web UI、評価モジュール、SSE progress events
3. **RAG 精度**: メタデータフィルタリング未実装のため、約款の電圧種別/エリア混在が発生しうる
