## 概要

Zendesk問い合わせ Tier 2（手続き系42件）を分析した結果、76%（32件）がスタッフ対応必須の操作であることが判明。DGMセルフサービス分（10件）は既存チャンクで概ねカバー済みのため、スタッフ対応が必要な32件に対するZendeskエスカレーション機能のリッチ化を行う。

### 操作分類サマリー

**DGMセルフサービス（10件）** — 既存チャンクでカバー済み、追加対応は最小限:
- 請求書送付先メールアドレスの変更（5件）
- 請求書の確認・ダウンロード（4件）
- 請求書の再発行（1件）

**スタッフ対応が必要（32件）** — 本タスクの対象:
- 口座振替依頼書の提出方法（10件）
- 口座振替・支払方法の変更手続き（10件）
- DGMアカウント発行・再発行（9件）
- DGMログイン初回設定メール再送（2件）
- 名義変更の手続き（1件）

詳細分析: `docs/superpowers/specs/2026-03-22-tier2-operation-classification.md`

### ST-1: DGMセルフサービスのナレッジ補強

**意図**: DGMで自己解決できる操作は、チャットボットが手順案内して完結させる。既存チャンクの不足分のみ補強。

**対象（10件）**:
- 請求書送付先メールアドレスの変更 → `DGM操作/procedures.yaml` id:1 に既存。十分か確認
- 請求書の確認・ダウンロード → `DGM操作/procedures.yaml` id:3 に既存。十分か確認
- 請求書の再発行 → 同上でカバー

**作業内容**:
- 既存チャンクの回答がZendeskの実質問に対して十分か確認
- 不足があれば `DGM操作/procedures.yaml` にQ&Aを追記
- 想定作業量: 小（0-3チャンク追加程度）

### ST-2: エスカレーション事前ヒアリングフローの設計

**意図**: スタッフ対応が必要な32件について、チャットボットがエスカレーション前に必要情報を聞き出し、チケットの情報品質を向上させる。CSスタッフの「お客様に追加確認」のラウンドトリップを削減。

**カテゴリ別の必要ヒアリング項目**:

#### 口座振替依頼書の提出（10件）
- 契約番号 / 顧客番号
- 新規提出 or 不備再提出
- 送付先住所の確認（赤坂本社）

#### 口座振替・支払方法の変更（10件）
- 契約番号 / 顧客番号
- 現在の支払方法（口座引落 / 振込 / 請求書払い）
- 希望する変更内容
- 変更希望時期

#### DGMアカウント発行・再発行（9件）+ ログインメール再送（2件）
- 契約番号 / 顧客番号
- 登録メールアドレス（覚えている場合）
- 新規発行 or パスワード再設定 or 設定メール再送
- メールが届かない場合: 迷惑メールフォルダ確認済みか

#### 名義変更（1件）
- 契約番号
- 変更理由（M&A / 売却 / 法人合併 / その他）
- 新契約者の法人名

**設計の成果物**:
- カテゴリ別ヒアリング項目のJSON定義（プロンプトまたはツール設定に組み込む）
- エスカレーション時のチケット本文テンプレート

**対象ファイル**:
- `src/app/services/agent/tools/zendesk_escalation.py` — エスカレーションツールの拡張
- `src/app/services/agent/prompts/` — プロンプトへのヒアリングガイダンス追加

### ST-3: Zendeskチケット自動カテゴリ・タグ付与の実装

**意図**: エスカレーション時にチケットに適切なカテゴリ・タグを自動付与し、CSスタッフの仕分け作業を削減する。

**タグマッピング**:

| ユーザーの質問内容 | 自動付与タグ |
|------------------|------------|
| 口座振替依頼書の提出・郵送 | `請求__口座振替依頼書` |
| 支払方法の変更（引落→振込等） | `請求__支払方法変更` |
| DGMアカウント発行・パスワード再設定 | `システム__id／pass再発行` |
| DGMログイン設定メール再送 | `適宜業務__ミエルカ/dgmアイパス送付` |
| 名義変更 | `契約__名義変更` |

**実装方針**:
- LLMによる質問内容の分類 → タグ選択（既存のZendeskタグ体系に合わせる）
- `zendesk_escalation.py` のチケット作成時にタグを付与

**対象ファイル**:
- `src/app/services/agent/tools/zendesk_escalation.py`

---

### ST-4: テスト・目視レビュー

**意図**: エスカレーション改善が正しく動作するか確認する。

**テスト観点**:
- ヒアリングフローが適切に発動するか（手続き系の質問時）
- 収集した情報がチケット本文に正しく含まれるか
- タグの自動付与が正しいか
- DGMセルフサービスの質問では手順案内が返り、エスカレーションされないか

**テストケース例**:

| 質問 | 期待動作 |
|------|---------|
| 「口座振替依頼書の送付先を教えてください」 | 送付先住所を案内 + 必要に応じてエスカレーション |
| 「DGMのパスワードを忘れました」 | パスワードリセット手順を案内 → 解決しなければアカウント再発行でエスカレーション |
| 「支払方法を口座引落から振込に変更したい」 | 契約番号等をヒアリング → タグ `請求__支払方法変更` でチケット起票 |
| 「請求書をダウンロードしたい」 | DGM操作手順を案内（エスカレーションしない） |
| 「名義変更の手続きを教えてください」 | 必要書類を案内 + 変更理由をヒアリング → タグ `契約__名義変更` でチケット起票 |
